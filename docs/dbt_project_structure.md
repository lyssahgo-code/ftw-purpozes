# DBT Project Structure (Template / Reference)

**Purpose**: To serve as reference for my non-techy normie brain to remember how dbt projects are usually organized >,<

````text
dbt/
├─ transforms/                
│  ├─ 01_project_name/                # Rename as needed
│  │  ├─ logs/                  # Logs generated by dbt runs
│  │  │  └─ dbt.log
│  │  ├─ macros/                # SQL macros and reusable snippets
│  │  │  ├─ create_schema.sql
│  │  │  ├─ generate_database_name.sql
│  │  │  └─ generate_schema.sql
│  │  └─ models/                # Main dbt models
│  │     ├─ clean/              # Clean/standardized staging tables
│  │     │  └─ {{SQL files go here}}
│  │     ├─ mart/               # Dimensional/fact tables for analysis
│  │     │  └─ {{SQL files go here}}
│  │     └─ sources.yml         # Definitions for raw tables
├─ target/                      # DBT-generated output (compiled SQL, run artifacts)
├─ .user.yml                     # User-specific configurations (optional)
├─ dbt_project.yml               # Main dbt project config
├─ profiles.yml                  # DBT connection profiles
└─ dockerfile                    # If we run dbt in Docker
